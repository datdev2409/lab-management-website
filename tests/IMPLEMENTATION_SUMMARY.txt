🚀 HOW TO USE
═══════════════════════════════════════════════════════════════════════════════

Initial Setup:
  $ cd tests
  $ npm install
  $ npx playwright install chromium

Run All Tests:
  $ npm test

Run with UI (Interactive):
  $ npm run test:ui

Run in Headed Mode:
  $ npm run test:headed

Debug Tests:
  $ npm run test:debug

View Test Report:
  $ npx playwright show-report

Run Specific Test:
  $ npx playwright test playwright/specs/01-auth.spec.js

🔧 MAINTENANCE & UPDATES
═══════════════════════════════════════════════════════════════════════════════

Adding New Tests:
  1. Create test file in playwright/specs/
  2. Use existing helpers or create new ones
  3. Follow naming conventions (NN-feature.spec.js)
  4. Use unique test data (timestamps)
  5. Update documentation if needed

Updating Helpers:
  1. Modify helper function in playwright/helpers/
  2. Update all tests using the helper
  3. Test changes locally
  4. Document changes if significant

Debugging Failed Tests:
  1. Check console output for errors
  2. Review screenshots in test-results/
  3. Run test in debug mode: npm run test:debug
  4. Use page.pause() for step-through debugging
